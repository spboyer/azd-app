### 3. Log Table Selector ⚠️ **INCOMPLETE - NEEDS INTEGRATION**
**Location**: [docs/specs/log-table-selector/](../specs/log-table-selector/)  
**Status**: Components built but NOT integrated into LogsPane  
**NEXT Pointer**: Integration task needed

**Implementation Status**: Backend complete, frontend components built, integration missing
- ✅ Backend table discovery API, endpoints, query builder
- ✅ Frontend components (TableSelector, LogConfigPanel)
- ✅ Hooks (useLogConfig, useAvailableTables)
- ✅ HistoricalLogPanel has integration (but panel itself not used)
- ❌ **LogsPane missing Settings icon for Azure mode**
- ❌ **No way to open LogConfigPanel from regular streaming logs**

**The Design Intent**: Each LogsPane should have a Settings (cog) icon when in Azure mode that opens the LogConfigPanel, allowing users to configure which Log Analytics tables to query for that specific service.

**What Was Built**:
- Integration was added to HistoricalLogPanel (a separate advanced query panel)
- But HistoricalLogPanel itself is not rendered in the app
- Regular LogsPane (streaming logs) was never integrated

**What's Needed**:
1. Add Settings2 (cog) icon to LogsPaneHeader when `logMode === 'azure'`
2. Add state to LogsPane for opening/closing LogConfigPanel
3. Pass serviceName to LogConfigPanel for per-service configuration
4. Test flow: LogsPane (Azure mode) → Click Settings → TableSelector appears

**Current User Impact**: Users viewing Azure streaming logs cannot configure which tables are queried. Feature exists in code but has zero user accessibility.
